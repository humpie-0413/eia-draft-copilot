# 사용자 가이드

EIA Draft Copilot의 전체 사용 흐름을 단계별로 설명합니다.

## 전체 사용 흐름

```
1. 프로젝트 생성
2. 사업 부지 geometry 입력
3. 공공데이터 수집 (에어코리아, 수질 DB)
4. 증거(Evidence) 확인 및 관리
5. 유사사례 매칭 확인
6. 섹션 플래너로 충족도 확인
7. 초안 뼈대 확인
8. QA 검증
9. DOCX 문서 내보내기
```

---

## 1. 프로젝트 목록 (`/projects`)

### 기능
- 등록된 프로젝트 목록을 표시합니다.
- 각 프로젝트에 대해 다음 버튼이 제공됩니다:
  - **증거(Evidence)**: Evidence Workbench로 이동
  - **섹션**: 섹션 플래너로 이동
  - **초안**: 초안 뼈대 뷰어로 이동
  - **QA**: QA 결과 페이지로 이동
  - **유사사례**: 유사사례 매칭 페이지로 이동

### 프로젝트 생성
- API를 통해 프로젝트를 생성합니다 (현재 UI 폼은 미구현).
- 필수 입력: 프로젝트명
- 선택 입력: 설명, 사업유형, GeoJSON geometry

### 사업유형 (project_type)
| 코드 | 설명 |
|------|------|
| road | 도로 |
| railway | 철도 |
| power_plant | 발전소 |
| industrial | 산업단지 |
| housing | 주택단지 |
| airport | 공항 |
| port | 항만 |
| dam | 댐 |
| reclamation | 매립/간척 |
| other | 기타 |

---

## 2. Geometry 입력

프로젝트의 사업 부지 경계를 GeoJSON Polygon 또는 MultiPolygon으로 입력합니다.
좌표계는 EPSG:4326 (WGS84 경위도)입니다.

```json
{
  "type": "Polygon",
  "coordinates": [
    [[126.97, 37.56], [126.98, 37.56], [126.98, 37.57], [126.97, 37.57], [126.97, 37.56]]
  ]
}
```

Geometry는 다음 기능에 활용됩니다:
- 유사사례 매칭 시 위치 유사도 계산 (centroid 간 haversine 거리)
- 사업 면적 추정 (ST_Area)

---

## 3. Evidence Workbench (`/projects/[id]/evidences`)

### 기능
- 프로젝트에 연결된 모든 증거(Evidence) 데이터를 관리합니다.
- 증거 = 환경 지표의 실측 데이터 1건 (예: "PM10_연평균 = 42 ug/m3")

### 데이터 수집 버튼
1. 화면 상단의 **"데이터 수집"** 버튼을 클릭합니다.
2. 커넥터를 선택합니다:
   - **에어코리아 대기질**: 측정소명(필수), 조회기간(DAILY/MONTH/3MONTH) 입력
   - **국립환경과학원 수질 DB**: 측정연도(필수), 측정지점 코드(선택) 입력
3. **"수집 시작"** 버튼을 클릭하면 공공데이터 API에서 데이터를 가져옵니다.
4. 수집 완료 시 결과 요약(성공/실패, 증거 건수)이 표시됩니다.
5. 수집된 증거는 자동으로 목록에 반영됩니다.

### 필터
- **분야 필터**: 12개 환경 분야(대기질, 수질, 토양, 소음·진동 등) 중 선택
- **스크리닝 필터**: 본 평가 / 스크리닝 전용 데이터 분리

### 증거 수동 추가/편집
- **추가**: 분야, 지표명, 값, 단위, 관측일 등을 직접 입력
- **편집**: 기존 증거의 정보를 수정
- **삭제**: 확인 다이얼로그 후 삭제

### 증거 상세 보기
- 테이블의 **눈 아이콘**을 클릭하면 상세 시트가 열립니다.
- 메타데이터(JSON)와 원본 스냅샷(raw_payload) 확인 가능

---

## 4. 유사사례 매칭 (`/projects/[id]/similar-cases`)

### 기능
- 현재 프로젝트와 유사한 과거 환경영향평가 사례를 검색합니다.
- 4가지 차원의 가중 유사도를 계산합니다:
  - **사업유형** (35%): 같은 유형이면 1.0, 같은 그룹이면 0.5
  - **위치** (25%): centroid 간 거리 기반 (50km → 0.5)
  - **규모** (20%): 사업 면적 비교
  - **환경분야** (20%): 공통 환경 분야 비율 (Jaccard 유사도)

### 매칭 결과 테이블
- 유사도 점수(0~100%)별 색상 표시: 70%+ 녹색, 40~70% 황색, 40% 미만 회색
- 각 사례를 클릭하면 상세 정보(요약, 주요 발견사항 등)를 확인할 수 있습니다.

---

## 5. 섹션 플래너 (`/projects/[id]/sections`)

### 기능
- 환경영향평가서 11개 섹션의 증거 충족 상태를 확인합니다.
- 각 섹션 카드에 표시되는 정보:
  - **상태**: 비어 있음(empty) / 부분 충족(partial) / 완료(complete)
  - **충족도**: 진행률 바 (0~100%)
  - **필수 지표 목록**: 각 지표의 충족 여부 (✓ / ○)와 증거 건수

### 충족도 계산 방식
- `coverage_ratio = 충족된 필수 지표 수 / 전체 필수 지표 수`
- **screening_only=False**인 본 평가 데이터만 대상으로 계산합니다.

---

## 6. 초안 뼈대 (`/projects/[id]/draft`)

### 기능
- 수집된 증거를 기반으로 환경영향평가서 초안의 뼈대를 자동 생성합니다.
- 각 섹션별로 다음이 표시됩니다:
  - 섹션 제목 및 설명
  - 증거 건수 배지
  - 증거 테이블 (지표, 값, 단위, 관측일)
  - 자동 생성된 요약 텍스트

### 요약 텍스트 형식
```
[섹션명] 현황 근거 데이터 (N건)

■ PM10_연평균
  - 측정값: 42 ug/m3 (관측: 2024-01-15)
  - 측정값: 38 ug/m3 (관측: 2024-02-15)

■ PM2.5_연평균
  - 측정값: 21 ug/m3 (관측: 2024-01-15)
```

### QA 요약 바
- 초안 페이지 상단에 QA 요약이 통합 표시됩니다.
- critical / warning / info 이슈 건수와 Export 가능 여부

---

## 7. QA 결과 (`/projects/[id]/qa`)

### 기능
- 5개 결정적 QA 규칙을 실행하여 품질 이슈를 표시합니다.
- 이슈는 심각도별로 구분됩니다:
  - **critical** (빨간색): Export 차단 — 핵심 섹션 증거 부재, 필수 지표 누락 등
  - **warning** (노란색): Export 가능 — 비핵심 섹션 이슈, 충족도 부족 등
  - **info** (파란색): 참고 정보 — 단일 근거 지표 안내
- 각 이슈에 영향받는 지표 목록이 표시됩니다.

### 심각도 필터
- 특정 심각도만 필터링하여 볼 수 있습니다.

---

## 8. DOCX Export

### 기능
- QA 결과 페이지 또는 초안 뼈대 페이지에서 **"DOCX 다운로드"** 버튼을 클릭합니다.
- DOCX 파일이 브라우저를 통해 다운로드됩니다.

### Export Gate
- **critical 이슈가 1건이라도 있으면 Export가 차단됩니다.**
- 모든 critical 이슈를 해결(증거 추가)한 뒤 다시 시도해야 합니다.

### DOCX 구성
1. 표지 (프로젝트명, 생성일)
2. 목차
3. 섹션별 내용:
   - 섹션 제목 및 설명
   - 증거 테이블 (지표, 값, 단위, 관측일)
   - 요약 텍스트

---

## 주의사항

### screening_only의 의미
- `screening_only=True`: 스크리닝(사전 검토) 단계에서 수집한 참고용 데이터
- `screening_only=False`: 본 평가에 사용되는 정식 데이터
- **섹션 충족도 계산 및 초안 뼈대 생성에는 screening_only=False인 데이터만 사용됩니다.**

### critical 이슈와 Export 차단
- 핵심 4개 섹션(대기질, 수질, 소음·진동, 생태)에 증거가 전혀 없거나 필수 지표가 누락되면 critical 이슈가 발생합니다.
- critical 이슈가 있으면 DOCX Export가 차단됩니다.
- 해결 방법: 해당 섹션의 필수 지표에 대한 증거를 추가하세요.

### Evidence First 원칙
- 이 시스템은 **증거 기반(evidence-based)** 원칙으로 동작합니다.
- 초안 뼈대의 모든 내용은 수집된 증거에서 직접 도출됩니다.
- 증거 없이 자유 작문(LLM 생성)되는 내용은 없습니다 (unsupported claim 금지).
- 근거가 부족하면 초안에 해당 내용이 포함되지 않으며, QA에서 누락으로 표시됩니다.
